Linguagens de Programção

As primeiras linguagens utilizadas na história foram as Linguagens de Baixo Nível, que eram utilizadas na programação dos primeiros computadores. Considerada a primeira Linguagem de Programação Estruturada, porém ainda de baixo nível, o Assembly permitiu que os primeiros computadores fossem programados. # Linguagem de Baixo Nível: Mais próxima da Liguagem de Máquina # Linguagem de Alto Nível: Mais próxima da Linguagem Humana Cada linguagem de programação possui suas características, vantagens e desvantagens e podem ser utilizadas em diferentes cenários. As linguagens foram evoluindo com o passar do tempo e foram surgindo também as Linguagens de Alto Nível. No início existiam poucas Linguagens, mesmo que fossem de Alto Nível. Algumas das primeiras, mais conhecidas – até hoje – são: COBOL, FORTRAN, PASCAL e CLIPPER. Algumas das novas liguagens são: Python, Ruby, Swift, JavaScript, PHP, SQL, HTML e etc.

Paradigmas de Programação

Os paradigmas, classificam as linguagens, através de particularidades como suas funcionalidades e sua estrutura. De acordo com o paradigma de uma linguagem, o programador pode ter uma noção sobre como um software será abordado e como será a estrutura de desenvolvimento. De acordo com o Dicionáio Aurélio, paradigma é: Conjunto dos termos ou elementos que podem ocorrer na mesma posição ou contexto de uma estrutura. Uma Linguagem de Programação, porém, pode possuir mais de um paradigma e estes definirão como a linguagem deverá operar para resolver determinados problemas. Um dos paradigmas mais utilizados nos últimos anos é o paradigma de Orientado a Objetos (OO). Hoje em dia, existem diversos paradigmas em uso. Alguns dos mais comuns são: #Funcional #Procedural/Imperativo #Orientado a Objetos #Orientado a Eventos #Lógico #Declarativo

PARADIGMA FUNCIONAL

Paradigma voltado para desenvolvimento de funções, como na matemática. Pode ser utilizado por linguagens que não são definidas como Linguagens Funcionais. Atendem ao Paradigma Funcional: F#, Lisp, Erlang, Haskell, Elixir, Mathematica

PARADIGMA PROCEDURAL

Também conhecido como Paradigma Imperativo. Foca em trabalhar com procedimentos internos, onde não exista a criação de objetos e classes. Atendem ao Paradigma Procedural: C, C++, PHP, Perl, C#, Pascal, Ruby

PARADIGMA ORIENTADO A OBJETOS

OO é sua sigla. Largamente utilizado entre diversas linguagens que podem optar por utilizá-lo, ou não. Utiliza uma lógica que se aproxima bastante do mundo real, onde categorizamos objetos em classes. Atendem ao Paradigma Orientado a Objetos: Java, C#, PHP, Python, Ruby

PARADIGMA ORIENTADO A EVENTOS

Basicamente é utilizado por linguagens que possuam/utilizem interfaces gráficas onde ocorrerão eventos ativados pelo usuário. Em um formulário de preenchimento, por exemplo, pode existir um botão “Salvar” que ao ser clicado pelo usuário, acionará um evento para envio de informações ao BD. Atendem ao Paradigma Orientado a Objetos: Delphi, Visual Basic, C#, Python, Java

PARADIGMA LÓGICO

Paradigma bastante utilizado em aplicações voltadas para aplicação de Inteligência Artificial Uma de suas características fortes é a argumentação. E seus principais elementos são: Proposição, Regras de Interferência e Busca O Prolog é uma das linguagens que atende este paradigma. Exemplo de utilização do paradigma. Proposição: Chico é um gato. Regras de Interferência: Todo gato é um felino. Busca: Chico é um felino? Resultado: Se Chico é um gato e todo gato é felino, então Chico é um felino.

PARADIGMA DECLARATIVO

Este paradigma trabalha na descrição de o que será feito, e não de como será executado. Foi baseado nos paradigmas lógico e funcional Linguagens de marcação representam este paradigma. Atendem ao Paradigma Declarativo: HTML, XML, Prolog.

Usabilidade

A usabilidade é um termo utilizado para definir a facilidade com que as pessoas empregam uma ferramenta ou mesmo um objeto para realizar uma tarefa. No ambiente de software, os sistemas são as ferramentas utilizadas para a execução de tarefas pelo usuário e encontramos na usabilidade o momento do dialogo entre o usuário e a interface do software. A usabilidade é capacidade do sistema em fazer com que o usuário tenha sucesso na execução de suas tarefas. Fácil aprendizagem, utilização eficiente e gestão de erros são pontos fundamentais para que o usuário perceba a boa usabilidade. No desenvolvimento de um novo software especifico do qual já há experiências anteriores, a busca pela excelência na usabilidade torna-se facilitada Todavia, quando essa experiência é inexistente ou mesmo pouco avançada, o trabalho torna-se mais difícil e a busca pela boa usabilidade deve ser precedida de uma eficiente pesquisa de opinião, pesquisa em que muitas vezes o usuário não está disposto a fornecer ou mesmo não é algo claro para que possa ser transmitido por ele com facilidade. Independente disso, se o usuário não sentir clareza e facilidade de uso, simplesmente não vai mais utilizar ou mesmo criará empecilhos para a utilização do software e provavelmente procure outro fornecedor que atenda as suas necessidades. Assim é importante ter em mente que a usabilidade é um elemento norteador ao processo de desenvolvimento de software, devendo ser tratada com relevância. Diante do grande volume de informação existente atualmente, a usabilidade cada vez mais vem sendo tratada por diversas empresas como ponto estratégico para conquistar novos consumidores/cliente, pois o fato é: – Se um sistema é complicado de ser usado pelo usuário, para que existir o sistema? É necessário ter em mente que o desenvolvimento de um sistema não é para o programador ou para o analista utilizar, mas sim para tornar as tarefas dos usuários mais ágeis e eficientes, para isso é que as empresas de software são contratada
